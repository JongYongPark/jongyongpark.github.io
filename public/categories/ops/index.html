    How to send findings from SRM (Software Risk Management) to Azure DevOps Work Items

    SRM (Software Risk Management)의 결함 정보를 Azure DevOps Work Items로 전송하려면 Azure DevOps REST API를 사용하여 작업 항목(Work Item)을 생성하거나 업데이트할 수 있습니다. 아래는 이를 구현하는 방법에 대한 단계별 설명입니다.

    1. Azure DevOps API 준비

    • Azure DevOps Organization 및 프로젝트 설정: Azure DevOps 조직과 프로젝트가 이미 생성되어 있어야 합니다.
    • Personal Access Token (PAT) 생성:
      1. Azure DevOps 포털에 로그인합니다.
      2. 사용자 설정 > Personal Access Tokens로 이동합니다.
      3. 새로운 토큰을 생성하고, Work Items (Read & Write) 권한을 부여합니다.
      4. 토큰을 안전하게 저장합니다 (이 토큰은 API 호출 시 사용됩니다).

    2. Azure DevOps Work Item API 엔드포인트

    • Azure DevOps REST API를 사용하여 작업 항목을 생성하거나 업데이트할 수 있습니다.
    • Work Item 생성 API: 
      POST https://dev.azure.com/{organization}/{project}/_apis/wit/workitems/${type}?api-version=7.1-preview.3
      • {organization}: Azure DevOps 조직 이름.
      • {project}: 프로젝트 이름.
      • {type}: 작업 항목 유형 (예: Bug, Task, Issue 등).

    3. SRM 결함 정보를 Azure DevOps Work Item으로 변환

    • SRM에서 제공하는 결함 정보를 Azure DevOps Work Item 필드에 매핑합니다.
    • 예를 들어:
      • SRM 결함 제목 → Azure DevOps Title
      • SRM 결함 설명 → Azure DevOps Description
      • SRM 결함 심각도 → Azure DevOps Severity
      • SRM 결함 상태 → Azure DevOps State
